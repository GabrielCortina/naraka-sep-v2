## Layout Contract

### Desktop (>=640px)

```
+--------------------------------------------------+
| Usuarios                        [+ Novo Usuario] |
+--------------------------------------------------+
| Nome        | Role       | Status  | Acoes       |
|-------------|------------|---------|-------------|
| Ana Silva   | admin      | Ativo   | [edit] [sw] |
| Carlos M.   | separador  | Ativo   | [edit] [sw] |
| Pedro L.    | fardista   | Inativo | [edit] [sw] |
+--------------------------------------------------+
```

- Page title "Usuarios" at 20px semibold, left-aligned
- "+ Novo Usuario" button at top-right, primary variant (black bg, white text)
- Table fills available width inside the authenticated layout content area
- Table header row: muted background, 12px semibold uppercase text
- Table rows: 14px regular, vertical padding 12px (sm + xs), hover state with muted background
- Acoes column: Pencil icon button (edit), Switch component (ativo toggle)
- Sort: alphabetical by nome ascending (default)

### Mobile (<640px)

```
+-----------------------------+
| Usuarios                    |
| [+ Novo Usuario]            |
+-----------------------------+
| +-------------------------+ |
| | Ana Silva         admin | |
| | Ativo       [edit] [sw] | |
| +-------------------------+ |
| +-------------------------+ |
| | Carlos M.    separador  | |
| | Ativo       [edit] [sw] | |
| +-------------------------+ |
```

- Page title and CTA button stack vertically (title on top, button full-width below)
- Each user renders as a card (border, rounded-md, padding 16px)
- Card layout: top row = nome (14px semibold) + role badge; bottom row = status badge + action icons
- Cards separated by 8px gap

## Interaction States

### Loading
- Skeleton rows: 5 rows of skeleton placeholders (shimmer animation) matching table column widths
- On mobile: 3 skeleton cards
- Loading triggered on initial page mount

### Empty state
- Shown when users table has 0 rows (unlikely in practice since admin exists)
- Centered in table area

### Modal states
- **Create mode:** Title "Novo Usuario", all fields empty, PIN required, Save button disabled until nome + PIN + PIN confirmation + role are filled
- **Edit mode:** Title "Editar Usuario", nome and role pre-filled, PIN fields empty with placeholder "Deixe vazio para manter atual", Save button enabled immediately (PIN optional)
- **Saving:** Save button shows spinner, all fields disabled
- **Validation errors:** Inline red text below field. PIN mismatch: "PINs nao coincidem". Nome empty: "Nome obrigatorio". PIN length: "PIN deve ter 4-6 digitos"

### Toggle ativo/inativo
- **Deactivating (ativo -> inativo):** Opens DeactivateDialog with confirmation copy
- **Reactivating (inativo -> ativo):** Immediate toggle, no confirmation. Success toast
- **Toggle disabled during API call:** Switch shows loading state (reduced opacity)

### Error handling
- API errors: Sonner toast with destructive variant, message from API or generic fallback
- Network errors: "Erro de conexao. Tente novamente."

## Accessibility Notes

- Modal traps focus (handled by Radix Dialog)
- Switch component has aria-label: "Ativar/desativar {nome}"
- Edit button has aria-label: "Editar {nome}"
- Table uses semantic `<table>` markup (shadcn Table component)
- PIN fields use `inputMode="numeric"` and `type="password"` (masked input, numeric keyboard on mobile)
- Deactivate dialog uses `AlertDialog` pattern (requires explicit action, not dismissable by clicking outside)
