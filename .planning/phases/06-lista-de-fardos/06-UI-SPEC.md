| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, badge internal padding |
| sm | 8px | Gap between SKU and codigo IN text, compact element spacing |
| md | 16px | Fardo item internal padding, filter bar gaps |
| lg | 24px | Space between fardo items, section padding |

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Codigo IN text, endereco text, filter labels |
| Label | 12px | 400 (regular) | 1.4 | "CONTEM" label, "ID:" prefix, status badges, counter labels |
| Heading | 16px | 600 (semibold) | 1.3 | SKU display on each fardo item, page title |
| Display | 24px | 600 (semibold) | 1.2 | Quantity number (large number next to "CONTEM") |

Weight rationale: 2 weights only (400 regular + 600 semibold). Label at 12px is visually distinct from Body at 14px without needing a separate weight. Display at 24px provides sufficient size contrast over Heading at 16px without needing 700 bold.

Source: D-03 specifies "SKU em bold grande", "numero grande bold" for quantity, "texto menor cinza" for ID and labels.

## Component Inventory

### FardoItem (each fardo row)

```
+--+-----------------------------------------------+------------+-------+
|  | SKU-EXAMPLE-001              [16px semibold]   | CONTEM     |  OK   |
|CB| ID: IN04                     [14px gray]       | 120        |  N/E  |
|  | pin 16PAREDE                 [14px, pin icon]  | [24px semi]|       |
|  | [Badge: Encontrado]          (conditional)     |            |       |
+--+-----------------------------------------------+------------+-------+
   ^--- blue left border 3px
```

- Card: white bg, rounded-lg (var(--radius)), border-l-[3px] border-blue-500
- Shadow: shadow-sm (subtle elevation over gray bg)
- Min height: 80px
- Horizontal layout with flexbox, items-center
- Checkbox: left side, 20px visual, 44px tap zone (lider/admin only, hidden for fardista per D-05)
- Status backgrounds: conditional bg-green-50 / bg-red-50 per D-19/D-22
- Buttons disabled + opacity-40 when status is encontrado or nao_encontrado (D-03)
- Spinner replaces button icon while API processes (D-30)

### FardoFilters (header controls)

```
[Search: Buscar codigo IN...]  [Pendentes|Encontrados|N/E|Todos]  [Atribuicao v]  [Ordenar v]
```

- Search input: shadcn Input with Search icon (lucide), 300ms debounce (Claude's discretion)
- Status filter: Tab-style chips using shadcn Badge variant=outline for unselected, variant=default for selected. Four options: Pendentes / Encontrados / N/E / Todos
- Attribution filter: shadcn-style dropdown or segmented control: Todos / Atribuidos / Nao Atribuidos
- Sort: dropdown with 3 options — Endereco (A-Z) default, SKU (A-Z), Importacao

### FardoCounters (summary bar)

```
  23 pendentes  |  8 encontrados  |  3 N/E
```

- Inline horizontal, separated by vertical pipes or dots
- Numbers in semibold (600), labels in regular weight (400)
- Update in realtime via Supabase subscriptions (D-29)

### SelectionBar (floating footer)

```
+---------------------------------------------------------------+
|  5 fardos selecionados        [Imprimir]  [Atribuir]         |
+---------------------------------------------------------------+
```

- Fixed bottom, full width, z-50
- Background: var(--primary) (dark), text: var(--primary-foreground) (white)
- Appears with slide-up animation when checkboxes selected (D-23)
- Hidden when 0 selected
- Buttons: ghost variant on dark background (white text, hover with white/10 bg)
- Left: count text. Right: action buttons.
- Only visible for lider/admin (D-25)

### Page Header

```
Lista de Fardos                   [Sincronizar Estoque] [Selecionar Todos] [Imprimir]
23 pendentes | 8 encontrados | 3 N/E
[Search] [Filters]
```

- "Sincronizar Estoque" button: outline variant, RefreshCw icon (lucide). Lider/admin only (D-12)
- "Selecionar Todos" button: outline variant, CheckSquare icon. Lider/admin only (D-10)
- "Imprimir" button: outline variant, Printer icon (lucide). Prints selected or all (D-26)

## Interaction States

### FardoItem States

| State | Visual | Buttons | Source |
|-------|--------|---------|--------|
| Pendente | White bg, blue left border | OK (green) + N/E (red) enabled | D-03 |
| Processing OK | White bg, OK button shows spinner | Both buttons disabled | D-30 |
| Processing N/E | White bg, N/E button shows spinner | Both buttons disabled | D-30 |
| Encontrado | Green-50 bg, badge "Encontrado" | Both buttons disabled, opacity-40 | D-19 |
| Nao Encontrado | Red-50 bg, badge "Nao Encontrado" | Both buttons disabled, opacity-40 | D-22 |
| Selected (checkbox) | White bg + ring-2 ring-blue-500 | Normal | Selection mode |

### Transitions

- Pendente to Encontrado: background cross-fade 300ms ease, badge fade-in 200ms (D-29)
- Pendente to Nao Encontrado: background cross-fade 300ms ease, badge fade-in 200ms (D-29)
- Toast appears after transition: "Fardo {codigo_in} encontrado" or "Fardo {codigo_in} nao encontrado" (D-29)
- SelectionBar: translateY slide-up 200ms ease when count > 0, slide-down when count = 0

### Double-Click Prevention

- On OK/N/E click: immediately disable both buttons on that row + show spinner on clicked button (D-30)
- Re-enable only on API error (revert to pendente state)

## Responsive Behavior

| Breakpoint | Layout | Source |
|------------|--------|--------|
| Mobile (<768px) | Single column, full-width fardo items. Filters stack vertically. Search full width, chips wrap. Buttons stack if needed. | UIUX-02 mobile first for fardistas |
| Desktop (>=768px) | Single column with max-width constraint (768px centered). Filters in single row. | Fardos is inherently a list, not multi-column |

- Fardo items are always full-width within their container
- On mobile: OK and N/E buttons may stack vertically if horizontal space is insufficient, but prefer horizontal layout with smaller buttons
- Floating selection bar: full width on both mobile and desktop

## Accessibility Notes

- OK and N/E buttons must have aria-label: "Marcar fardo {codigo_in} como encontrado" / "Marcar fardo {codigo_in} como nao encontrado"
- Status badges use aria-live="polite" for screen reader announcement on status change
- Checkbox group uses aria-label="Selecionar fardo {codigo_in}"
- Keyboard: Tab navigates between fardo items, Enter/Space activates focused button
- Color is never the only indicator of status — badges with text always accompany color changes
