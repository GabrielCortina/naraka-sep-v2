## Interaction Contracts

### Action: Confirmar (full quantity)

| Step | What Happens | Visual Feedback |
|------|-------------|----------------|
| 1 | Separador taps green Check button on item | NumpadPopup opens |
| 2 | Separador enters quantity = quantidade_necessaria | Numpad shows quantity |
| 3 | Separador confirms | API call to /api/cards/progress. Item left border turns green. Check circle appears. Status: separado |

### Action: Parcial (quantity < needed, triggers cascade)

| Step | What Happens | Visual Feedback |
|------|-------------|----------------|
| 1 | Separador taps green Check button on item | NumpadPopup opens |
| 2 | Separador enters quantity < quantidade_necessaria | Numpad shows quantity |
| 3 | Separador confirms | Check/X buttons replaced by Loader2 spinner (animate-spin, h-4 w-4, text-muted-foreground). All buttons for this item disabled |
| 4a | Cascade finds bale(s) | Spinner stops. PARCIAL badge appears. New line "AGUARDAR FARDISTA" appears for remainder (blocked, amber badge, zinc border). Toast: "Fardo alternativo reservado -- AGUARDAR FARDISTA". In fardos list: new fardo with CASCATA badge appears unassigned |
| 4b | Cascade finds no bale | Spinner stops. Partial quantity confirmed. Remainder removed from card total (D-18). Item fades out if fully handled by transformacao. Toast: "Sem fardo disponivel -- enviado para Transformacao" |

### Action: NE (not found, triggers cascade)

| Step | What Happens | Visual Feedback |
|------|-------------|----------------|
| 1 | Separador taps red X button on item | Check/X buttons replaced by Loader2 spinner immediately. No confirmation dialog (D-03) |
| 2a | Cascade finds bale(s) | Spinner stops. Item status changes to AGUARDAR FARDISTA (blocked, amber badge, zinc border). Toast: "Fardo alternativo reservado -- AGUARDAR FARDISTA" |
| 2b | Cascade finds no bale | Spinner stops. Item fades out (opacity-0 over 300ms, then removed). Toast: "Sem fardo disponivel -- enviado para Transformacao" |

### Action: Search by SKU

| Step | What Happens | Visual Feedback |
|------|-------------|----------------|
| 1 | User types in search field | 300ms debounce before filtering |
| 2 | Filter applied | KanbanBoard shows only cards containing matching SKU. Empty columns hidden. Counters update to reflect filtered view |
| 3 | User clears search | Full kanban restored |

### Action: Leader cascade toast (realtime)

| Step | What Happens | Visual Feedback |
|------|-------------|----------------|
| 1 | Cascade creates new fardo with is_cascata=true | Realtime subscription fires |
| 2 | If user role is lider or admin | toast.info("Novo fardo de cascata adicionado -- atribuir fardista") via sonner |

## State Management

| State | Source | Updates Via |
|-------|--------|------------|
| Cards list | useCardData hook (Supabase query) | Realtime subscription on progresso, atribuicoes, reservas, trafego_fardos, transformacoes |
| Search term | Local useState in prateleira-client | User input with 300ms debounce |
| Spinner per item | Local useState Map<sku, boolean> in prateleira-client | Set true before cascade API call, false after response |
| Selected card | Local useState in prateleira-client | User tap on card |
| Modal open | Local useState in prateleira-client | Open/close events |
