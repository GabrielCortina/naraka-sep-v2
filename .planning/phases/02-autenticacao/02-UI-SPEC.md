## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | hsl(0 0% 3.9%) — near-black | Login page background, app shell background |
| Secondary (30%) | hsl(0 0% 100%) — white | Login card surface, content area, sidebar background |
| Accent (10%) | hsl(0 0% 9%) — primary black | Primary CTA button, active nav indicator |
| Destructive | hsl(0 84.2% 60.2%) — red | Error message text only |

**Rationale:** CONTEXT.md D-05 specifies "fundo escuro/neutro" for login with "card branco centralizado". The app uses the shadcn neutral palette already configured in globals.css. This phase uses dark mode for the login page background and light mode for the card surface and authenticated app shell.

**Accent reserved for:**
- "Entrar" button (primary CTA on login)
- Active tab indicator in bottom navigation
- Active sidebar item highlight

**Not used in this phase (future phases):**
- Marketplace colors (Shopee #ee4d2d, ML #ffe600, TikTok #25F4EE, Shein #000000)

## Component Inventory

### Login Page (`/login`)

| Element | Component | Behavior |
|---------|-----------|----------|
| Page background | Full viewport, dark (background in dark mode) | Centered flex container |
| Logo block | Text "NARAKA" (Display) + "SEP v2" (Label, muted) | Static, centered above card |
| Login card | shadcn Card | White surface, rounded-lg, shadow-lg, max-width 400px, width 100% with 16px horizontal margin on mobile |
| Nome field | shadcn Input + Label | Text input, placeholder "Seu nome", autocomplete="username" |
| PIN field | shadcn Input + Label | type="password" with inputMode="numeric" pattern="[0-9]*" maxLength={6}, placeholder "PIN", autocomplete="current-password" |
| Error message | Inline text below form | 14px, destructive color, only visible after failed attempt |
| Submit button | shadcn Button (default variant, full width) | Label: "Entrar", disabled state during loading |
| Loading state | Button with spinner | "Entrar" text replaced by Lucide Loader2 icon (animate-spin) + "Entrando..." |

**Login interaction flow:**
1. User types nome in first field
2. User types PIN in second field (numeric keyboard on mobile via inputMode)
3. User taps "Entrar" or presses Enter
4. Button enters loading state (disabled, spinner)
5. On success: redirect to role-specific default page (D-08)
6. On failure: show "Nome ou PIN incorreto" below form (D-06), re-enable button
7. Error clears when user modifies either field

## Accessibility

| Concern | Implementation |
|---------|----------------|
| Form labels | Every input has an associated `<label>` via shadcn Label component |
| Focus management | After failed login, focus returns to nome field |
| Keyboard | Enter key submits form from any field |
| PIN input | inputMode="numeric" for mobile numeric keyboard, type="password" to mask digits |
| Color contrast | White text on near-black background (21:1), black text on white card (21:1) — both exceed WCAG AAA |
| Screen reader | Error message uses role="alert" aria-live="polite" |
| Touch targets | All interactive elements minimum 44px height on mobile |
