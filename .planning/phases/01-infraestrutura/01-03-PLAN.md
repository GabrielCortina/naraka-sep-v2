
2. [BLOCKING] Aplicar migration no Supabase remoto (per schema_push_requirement):
   O usuario precisa ter um projeto Supabase criado e as env vars configuradas em .env.local.

   Linkar projeto Supabase:
   ```bash
   npx supabase link --project-ref <PROJECT_REF>
   ```
   O PROJECT_REF vem da URL do projeto Supabase (ex: abcdefghijklmnop).

   Aplicar migration:
   ```bash
   npx supabase db push
   ```

   NOTA: Se o comando pedir confirmacao interativa, pode ser necessario usar `echo "y" | npx supabase db push` ou configurar SUPABASE_ACCESS_TOKEN como env var para modo non-interactive. Se nenhum workaround funcionar, marcar para intervencao manual.

   ```bash
   npx supabase db inspect --linked
   ```
   Ou verificar via Supabase Dashboard -> Table Editor.

3. Aplicar seed data (usuario admin dev):
   ```bash
   npx supabase db execute --file supabase/seed.sql
   ```
   Se o comando nao existir nesta versao do CLI, usar o SQL Editor no Supabase Dashboard para executar o conteudo de seed.sql.

4. Gerar tipos TypeScript atualizados a partir do schema real (per D-06):
   ```bash
   npx supabase gen types typescript --project-id <PROJECT_REF> > src/types/database.types.ts
   ```
   Isto substitui o placeholder criado no Plan 02 pelos tipos reais auto-gerados.

5. Rebuild para verificar que os tipos gerados compilam:
   ```bash
   npm run build
   ```

6. Commit dos tipos gerados:

   ```bash
   git add src/types/database.types.ts

   ```
