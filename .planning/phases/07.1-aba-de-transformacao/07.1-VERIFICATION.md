#### 1. Mobile Bottom Tab Navigation (D-03)

**Test:** Open the app on a mobile viewport (or Chrome DevTools mobile emulation) as a separador or lider. Navigate to any authenticated page.
**Expected:** "Transformacao" tab appears in the bottom navigation bar with the Repeat icon, positioned after "Prateleira". Fardista role should NOT see the tab.
**Why human:** Bottom tab rendering depends on viewport breakpoints (`md:hidden` class) — programmatic verification cannot confirm rendering in mobile viewport.

#### 2. Card Animation to CONCLUIDOS Section (D-21)

**Test:** Log in as separador with an assigned transformacao card. Open the card modal, enter the exact required quantity via numpad, confirm.
**Expected:** A green toast "Transformacao concluida: [SKU]" appears. The card disappears from the active kanban column and reappears inside the CONCLUIDOS collapsible section at the bottom. Any card movement transition/animation (if present) is smooth.
**Why human:** The card movement happens reactively through state updates (realtime refetch → filteredCards recalculation → React re-render). There is no explicit CSS animation on the card wrapper for this transition (only `transition-opacity duration-300` on modal item rows). Visual behavior and the existence/quality of any animation requires a live browser session.

#### 3. Realtime Toast for Lider on New Card (D-13)

**Test:** Open two browser sessions simultaneously — one logged in as lider, another as separador or admin. In the second session, navigate to Prateleira and trigger a cascata action (mark an item as NE or Parcial when no bale is available) so that an item is sent to transformacao.
**Expected:** The lider session displays a `toast.info` notification with text containing "Nova transformacao criada:" and the card_key details, without requiring any page interaction or refresh.
**Why human:** Real-time cross-session toast behavior requires simultaneous active browser sessions with live Supabase realtime connections.
