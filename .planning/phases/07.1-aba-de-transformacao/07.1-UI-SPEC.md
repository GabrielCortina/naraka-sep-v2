## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge internal padding |
| sm | 8px | Compact element spacing, gap between counter items |
| md | 16px | Default element spacing, card content padding (p-4) |
| lg | 24px | Section padding, header vertical/horizontal padding |
| xl | 32px | Layout gaps between kanban columns |
| 2xl | 48px | Major section breaks, numpad button touch targets (h-12) |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets on numpad buttons are 48px (h-12) for mobile usability.

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (text-sm) | 400 (normal) | 1.5 |
| Label | 11px (text-[11px]) | 700 (bold) | 1.3 |
| Heading | 16px (text-base) | 700 (bold) | 1.2 |
| Display | 24px (text-2xl) | 700 (bold) | 1.2 |

Notes:
- Display used inside NumpadPopup for the quantity value and for SKU text inside modal
- Label used for badge text (TRANSFORMACAO badge, counter labels), matching existing text-[11px] pattern
- Counter numbers in header use text-base font-bold (Heading role), matching PrateleiraHeader
- Only 2 weights in use: 400 (normal) for body text and 700 (bold) for emphasis/headings

| Dominant (60%) | hsl(0 0% 100%) / --background | Page background, modal background |
| Secondary (30%) | hsl(0 0% 96.1%) / --secondary | Card backgrounds (bg-white), muted text, header border-b |

### Phase-Specific Color: Transformacao Badge

New CSS variable to add in globals.css:

```
--transformacao: 270 60% 65%;
```

Badge styling: `bg-purple-100 text-purple-800 border-purple-200` (light mode).
This follows the same pattern as the existing AGUARDAR FARDISTA badge (`bg-amber-100 text-amber-800 border-amber-300`).

Accent reserved for: TRANSFORMACAO badge on cards (top-right corner), transformacao status indicator on card border-left (border-l-purple-500).

### Existing Colors Used (no changes)

- Marketplace colors: shopee, ml, tiktok, shein (MarketplaceBadge on cards)
- Urgency colors: urgency-overdue, urgency-warning, urgency-ok (UrgencyBadge on cards)
- Green-600: Confirm button in numpad, success check icon, success toast
- Red-600: Error toast background (quantity mismatch)
- Blue-600: Assigned user indicator

## Modal Visual Contract: TransformacaoModal

```
+------------------------------------------+
| Shopee Xpress - KIT #1 -- Transf. #2    |
| [======progress bar==========]           |
| 75%                        3/4 pecas     |
+------------------------------------------+
| | SKU-ABC-123                            |
| | 0/15                                   |
| |                         Pegar [15] [v] |
+------------------------------------------+
| | SKU-DEF-456                            |
| | 0/8                                    |
| |                         Pegar [8]  [v] |
+------------------------------------------+
```

Key differences from ItemModal:
- SKU text uses text-2xl (24px) font-bold (Display role)
- Only ONE action button per item: green confirm (Check icon). No red NE button. No partial.
- NumpadPopup validation: quantity MUST equal quantidade_necessaria exactly. If mismatch, toast error "Quantidade deve ser exatamente {N}" and numpad stays open.
- No AGUARDAR FARDISTA badge (not applicable in transformacao context)
- No reservas display (no fardo reservas for transformacao items)
- Realtime updates: new items appear in the list without closing modal (D-10)
- Left border colors: blue-500 (pendente), green-500 (concluido)

## Interaction Contracts

### I-01: Open Card Modal

| Property | Value |
|----------|-------|
| Trigger | Tap/click on TransformacaoCard |
| Action | Open TransformacaoModal with all transformacao items for that card_key + numero_transformacao |
| Animation | slide-in-from-bottom (existing Dialog pattern) |

### I-02: Confirm Quantity (strict)

| Property | Value |
|----------|-------|
| Trigger | Tap green check button on item row |
| Action | Open NumpadPopup |
| Validation | Entered quantity MUST equal quantidade_necessaria exactly |
| Success | Toast green "Transformacao concluida: {SKU}" + item row shows green check |
| Error | Toast destructive "Quantidade deve ser exatamente {N}" + numpad stays open |
| Loading | Loader2 spinner replaces buttons while API call in flight |

### I-03: Assign Separador

| Property | Value |
|----------|-------|
| Trigger | Tap user icon on card |
| Accessibility | aria-label="Atribuir separador" on the icon button; tooltip "Atribuir separador" on hover (desktop) |
| Action | Open AssignModal with filterRole='separador' |
| Result | User name appears next to icon in text-blue-600, text-[11px] font-bold |

### I-04: Search by SKU

| Property | Value |
|----------|-------|
| Trigger | Type in search input |
| Debounce | 300ms |
| Action | Filter cards showing only those containing matching SKU in transformacao items |
| Empty result | Cards area shows nothing; counters still reflect full data |

### I-05: Realtime Item Arrival (D-10)

| Property | Value |
|----------|-------|
| Trigger | Supabase INSERT on transformacoes table for current card_key |
| Action | New item appears in open modal list without closing |
| Animation | Fade-in (opacity 0 to 1, 300ms) |

### I-06: New Sequential Card Creation (D-11)

| Property | Value |
|----------|-------|
| Trigger | New transformacao item arrives for a card_key whose current transformacao card is completed |
| Action | New card appears in kanban column with incremented numero_transformacao |
| Feedback | Toast for lider: "Nova transformacao criada: {grupo_envio} {tipo} #{importacao} -- Transformacao #{N}" |

### I-07: Card Completion

| Property | Value |
|----------|-------|
| Trigger | All items in transformacao card confirmed |
| Action | Card moves to CompletedSection |
| Animation | Card fades out from column (opacity 1 to 0, 300ms), appears in CompletedSection |
| Feedback | Toast green "Transformacao concluida" |
