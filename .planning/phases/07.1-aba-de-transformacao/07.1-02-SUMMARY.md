## Decisions Made

1. **Inline kanban layout**: Built kanban columns directly in transformacao-client.tsx instead of modifying shared KanbanBoard/KanbanColumn components, avoiding coupling and regression risk.
2. **CompletedSection cardRenderer**: Added optional prop with backward-compatible default. Prateleira continues using OrderCard unchanged.
3. **Strict numpad validation**: D-06 enforced -- quantity !== item.quantidade triggers toast.error, not accepted.
4. **Separate realtime channel**: Uses 'transformacao-realtime' channel, distinct from 'cards-realtime', to avoid interference.
