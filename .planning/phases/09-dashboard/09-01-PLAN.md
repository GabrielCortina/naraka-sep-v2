    Functions:
    - `computeResumo(pedidos, progresso, atribuicoes, transformacoes, now?)`: Groups pedidos by card_key, calculates progress per card using `calcProgress` from card-utils (including transformacao deduction). Uses `getUrgencyTier` to determine overdue. Returns `{ pecas_separadas, listas_pendentes, listas_concluidas, listas_em_atraso }`. "listas" = cards. A card is "concluida" when percent===100. A card is "em_atraso" when urgency==='overdue' AND percent<100. A card is "pendente" when percent<100 AND NOT overdue.
    - `computeProgressao(pedidos, progresso, transformacoes, now?)`: Groups by grupo_envio, sums all pecas and pecas_separadas (with transformacao deduction per card), computes percent, gets urgency tier per grupo_envio (based on combined percent), calculates deadline_ms from DEADLINES config. Returns `ProgressaoMetodo[]` sorted by COLUMN_ORDER from deadline-config.
    - `computeTopSeparadores(atribuicoes, pedidos, progresso, transformacoes)`: For each unique separador (atribuicoes.tipo==='separador'), sum pecas_separadas across their assigned cards and count cards where percent===100. Returns `RankingEntry[]` sorted by pecas_separadas desc. Position is 1-based.
    - `computeTopFardistas(baixados)`: Group by baixado_por, count fardos per user. Returns `RankingEntry[]` sorted by fardos_confirmados desc. Position is 1-based. pecas_separadas and cards_concluidos are 0 for fardistas.
    - `computeStatusFardos(trafego, baixados)`: PENDENTES = trafego WHERE status==='pendente'. ENCONTRADOS = trafego WHERE status==='encontrado' AND codigo_in NOT IN baixados.map(b=>b.codigo_in). BAIXADOS = baixados.length. Returns `{ pendentes, encontrados, baixados }`.
    - `computePorSeparador(atribuicoes, pedidos, progresso, transformacoes)`: For each separador, compute progress across assigned cards. Returns `SeparadorProgress[]` sorted by percent desc.

    All functions must use `Array.from(Map)` pattern for Map iteration (tsconfig compat, project convention).
