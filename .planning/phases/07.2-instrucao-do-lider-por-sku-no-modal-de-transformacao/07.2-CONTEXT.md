# Phase 07.2: Instrucao do lider por SKU no modal de transformacao - Context

**Gathered:** 2026-04-08
**Status:** Ready for planning
**Source:** User-provided decisions (inline)

<domain>
## Phase Boundary

Adicionar botao de instrucao do lider em cada linha de SKU no modal de transformacao existente. Lider/admin pode selecionar uma instrucao por SKU que aparece como badge colorido para todos os usuarios (incluindo separador) em tempo real via Supabase realtime. Zero mudanca em logica existente — apenas adicionar botao e badge.

</domain>

<decisions>
## Implementation Decisions

### Funcionalidade do botao
- **D-01:** Botao de instrucao visivel apenas para lider e admin em cada linha de SKU no modal de transformacao
- **D-02:** Botao abre popup com 4 opcoes de instrucao:
  - TRANSFORMACAO LIBERADA (verde)
  - SKU VAI CHEGAR (amarelo)
  - PEGAR NA VALERIA (azul)
  - PEGAR NA LOJA (roxo)
- **D-03:** Badge colorido aparece para todos (lider, admin e separador) apos selecionar instrucao
- **D-04:** Separador ve badge atualizar em tempo real via Supabase realtime (subscription ja existente no canal `transformacao-realtime`)

### Banco de dados
- **D-05:** Nova coluna `instrucao_lider` (text, nullable) na tabela `transformacoes`
- **D-06:** Valores validos: `null`, `TRANSFORMACAO_LIBERADA`, `SKU_VAI_CHEGAR`, `PEGAR_NA_VALERIA`, `PEGAR_NA_LOJA`

### API
- **D-07:** Novo endpoint `PATCH /api/transformacao/instrucao`
- **D-08:** Acesso restrito a roles `admin` e `lider` (separador e fardista bloqueados)
- **D-09:** Body: `{ transformacao_id: string, instrucao: string }`

### Realtime
- **D-10:** Subscription existente na tabela `transformacoes` para evento `*` (INSERT/UPDATE/DELETE) ja captura mudancas — badge atualiza automaticamente quando `instrucao_lider` muda

### Escopo
- **D-11:** Zero mudanca em logica existente — apenas adicionar botao e badge no modal existente

### Claude's Discretion
- Posicao exata do botao de instrucao na linha do SKU
- Design do popup de selecao (pode usar Popover do shadcn/ui)
- Tamanho e estilo exato do badge

</decisions>
