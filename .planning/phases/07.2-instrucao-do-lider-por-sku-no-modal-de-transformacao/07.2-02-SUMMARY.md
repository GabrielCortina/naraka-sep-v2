# Phase 07.2 Plan 02: Frontend instrucao do lider components Summary

Popover with 4 colored instruction options and badge display integrated into TransformacaoModal with role-based visibility (admin/lider edit, all see badges).

## What Was Built

1. **shadcn/ui Popover** (`src/components/ui/popover.tsx`): Installed via `npx shadcn@latest add popover`. Provides Popover, PopoverTrigger, PopoverContent exports based on @radix-ui/react-popover.

2. **InstrucaoBadge** (`src/features/transformacao/components/instrucao-badge.tsx`): Pure component rendering colored pill badge based on `InstrucaoLider` value. Returns null when instrucao is null. Color scheme: green (Liberada), yellow (Vai chegar), blue (Valeria), purple (Loja).

3. **InstrucaoPopover** (`src/features/transformacao/components/instrucao-popover.tsx`): Client component with Popover trigger (MessageSquare icon, 7x7 button) and 4 colored options. Calls `PATCH /api/transformacao/instrucao` with loading state and toast feedback. Supports toggle-off (clicking selected option sends `instrucao: null`). Trigger border color reflects current instruction.

4. **TransformacaoModal integration** (`src/features/transformacao/components/transformacao-modal.tsx`): Added `userRole: string` required prop. InstrucaoBadge rendered after SKU text for all users when `instrucao_lider` is set. InstrucaoPopover rendered in actions area only for admin/lider and only when item is not done. Zero changes to existing numpad/confirm/progress logic.

5. **Parent wiring** (`app/(authenticated)/transformacao/transformacao-client.tsx`): Passes `userRole` prop to TransformacaoModal from existing destructured prop.
